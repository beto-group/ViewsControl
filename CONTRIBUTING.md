# Contributing to Views Control

We welcome contributions from the community to help improve Views Control!

## How to Contribute

1. **Fork the Repository**: Create a personal fork of the repository on GitHub.
2. **Clone the Fork**: Clone your fork to your local development machine.
3. **Create a Feature Branch**: Create a branch for your work (e.g., `feature/amazing-new-mode`).
4. **Make Your Changes**: Write clean, modular code. Make sure to document any new features.
5. **Commit and Push**: Commit your changes with descriptive messages and push to your fork.
6. **Submit a Pull Request**: Open a Pull Request targeting the `main` branch of the official repository.

## Contribution Guidelines

- **Clean and Modular Code**: Follow the existing code style (modular ES6, pure JS/React, safe DOM mutations).
- **Graceful Fallbacks**: Ensure that any new external APIs or platform-specific modules have graceful fallback mechanisms for standard browsers.
- **No AI Contributors**: Under our repository governance guidelines, AI assistants, coding agents (e.g. Antigravity, Gemini, Copilot), or automated accounts cannot be credited as repository contributors or author entities.
- **Code of Conduct**: Treat all contributors and maintainers with respect.
